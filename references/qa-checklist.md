# QA Checklist

レポート出力前の最終確認項目。

## 必須チェック

- [ ] ナレーション全文を取得し、時系列に並べた
- [ ] ゲート軸（throughline）＋全軸に評価とコメントが付いている。条件付き軸（14 視覚同期 / 19 時間予算）は宣言時のみ評価し、未宣言ならスキップを明記
- [ ] 各評価に「具体的な根拠」と「改善案」、および**根拠となった原稿の一文の引用**を添えている（引用できない評価は要修正）
- [ ] スコアカードのテーブル（ゲート・構造・内容・伝達・説得・条件付き）が全て埋まっている
- [ ] 総合判定がゲート＋しきい値ルールに従っている（ゲート＝throughline/論理3/正確性5/網羅性6。正確性D→総合D、非正確性の単独DやthroughlineがC以下なら頭打ち）
- [ ] 「最大の強み」「最大の改善ポイント」「次のアクション」が記載されている
- [ ] 改善案が「もっと良くしましょう」ではなく「この文をこう変える」レベルの具体性

## 任意チェック

- [ ] `--source` 指定時、参照ソースとの差分テーブルが含まれる
- [ ] `--rewrite` 指定時、低評価セクションに書き直し文案が付いている
- [ ] Project Integration の `Visual Assets` が未設定なら、軸14をスキップした旨を明記
- [ ] Project Integration の `Audience` が宣言されているなら、軸10で参照したことを明示

## アンチパターン

|やってはいけないこと|理由|
|-|-|
|「面白いか」を構造ではなく主観で評価する|主観評価は再現性が低い。構造で判断できる「わかるか」「伝わるか」に集中する|
|軸5（正確性）の事実誤認を見逃す|致命的。1つでもあれば総合D|
|完璧を求めて全軸Aを目指す|A×12以上で「出して良い品質」と判定。過剰要求しない|
|抽象的な改善案で済ませる|「もっと具体的に」ではなく「この文をこう変える」を出す|
|恣意的な尺そのものを良し悪しにする|尺の長短自体は評価しない。ただし持ち時間が宣言されたら軸19で「収まるか」だけ見る|
|throughline を一文で言わずに評価を進める|中心メッセージを15語前後で言い切る（or 不在を指摘する）のがゲート。ここを飛ばさない|
|軸14のために全シーンを確認して時間を浪費する|サンプリング（各幕2-3シーン）で代用可|
