# Evaluation Axes (18 Axes)

各軸はA/B/C/Dの4段階で採点し、具体的な根拠と改善案を添える。
軸は4カテゴリに分類される。

## カテゴリA: 構造 (軸1-4)

### 軸1: フック（最初の30秒）

視聴者が「続きを見よう」と思うかどうか。

効果的なフックのパターン:

- **問いかけ**: 視聴者の経験に直接訴える（「〜したことありますか？」）
- **意外な数字**: 常識を覆すデータを提示する
- **ストーリー**: 具体的なエピソードから入る

|評価|基準|
|-|-|
|A|視聴者が即座に自分ごと化できる問いかけ or 意外性がある|
|B|興味を引くが、自分ごと化までは至らない|
|C|一般的な導入（「今日は〜について話します」）|
|D|フックなし、いきなり本題|

### 軸2: ナラティブアーク（SCQA構造）

全体がストーリーとして成立しているか。SCQAフレームワーク:

- **Situation（状況）**: 視聴者と共有できる前提
- **Complication（障害）**: 「でも実はこんな問題がある」という緊張
- **Question（問い）**: 視聴者が「どうすればいい？」と感じる瞬間
- **Answer（解答）**: 説得力ある解決策

特に「struggle（うまくいかなかったこと）」が共感と説得力を生む。成功談だけでは退屈。

|評価|基準|
|-|-|
|A|明確なSCQA + struggleが含まれている|
|B|SCQA構造はあるが、struggleが弱いか欠けている|
|C|問題→解決の流れはあるが、緊張感が薄い|
|D|情報の羅列で、ストーリーがない|

### 軸3: 論理の流れ

各セクションが前のセクションの上に積み重なっているか。

|評価|基準|
|-|-|
|A|全セクションが自然に繋がり、順序に必然性がある|
|B|概ね良いが、1-2箇所で唐突な転換がある|
|C|各セクション単体はOKだが、全体の流れが弱い|
|D|順序が恣意的、前後を入れ替えても違和感がない|

### 軸4: ペーシング（情報密度の配分）

|評価|基準|
|-|-|
|A|情報密度が全体で均一、息継ぎも適切|
|B|概ね良いが、一部で密度のムラがある|
|C|前半と後半で密度差が大きい|
|D|全体的に詰め込みすぎ or スカスカ|

## カテゴリB: 内容 (軸5-8)

### 軸5: 正確性

> [!IMPORTANT]
> 軸5だけは妥協しない。事実誤認は1つでもあれば指摘する。総合判定でも特別扱い。

|評価|基準|
|-|-|
|A|全て正確、出典付きデータと意見の区別も明確|
|B|概ね正確だが、1-2箇所で微妙な不正確さがある|
|C|重要でない部分に不正確さがある|
|D|核心部分に事実誤認がある|

### 軸6: 網羅性

参照ソースの重要情報がカバーされているか。

|評価|基準|
|-|-|
|A|コアメッセージを全てカバーし、過不足ない|
|B|主要な内容はカバーしているが、1-2点の補足が欲しい|
|C|いくつかの重要ポイントが欠けている|
|D|核心的な情報が欠落している|

### 軸7: 具体性

主張に具体例・数字・固有名詞が伴っているか。

|評価|基準|
|-|-|
|A|全主張に具体例・数字・固有名詞が伴い、解像度が高い|
|B|大部分は具体的だが、1-2箇所で抽象的な主張がある|
|C|具体例が散発的で、抽象論が多い|
|D|ほぼ抽象論で、根拠や実例がない|

### 軸8: 冗長性

|評価|基準|
|-|-|
|A|冗長な繰り返しなし、意図的な反復は効果的|
|B|1-2箇所で冗長だが、全体には影響しない|
|C|複数箇所で同じ内容を繰り返している|
|D|大幅にカットしても内容が変わらない|

## カテゴリC: 伝達 (軸9-14)

### 軸9: 明瞭さ

|評価|基準|
|-|-|
|A|全体的に平易で、抽象→具体の流れが一貫している|
|B|概ね明瞭だが、1-2箇所で理解に時間がかかる|
|C|部分的に難解で、具体例が不足している|
|D|全体的にわかりにくい|

口語では1文40文字を超えると聞き取りにくい。

### 軸10: 視聴者フィット

想定視聴者は Project Integration の `Audience` で宣言する。未宣言なら参照ソースから推定する。

|評価|基準|
|-|-|
|A|想定視聴者にぴったりで、初出用語は全て説明済み|
|B|概ね適切だが、1-2箇所で説明不足の専門用語がある|
|C|全体的にやや専門的すぎる or 基礎的すぎる|
|D|視聴者層が不明確、または大きくずれている|

### 軸11: エンゲージメント

飽きを防ぐ要素: 具体例 / データ / デモ / 比較 / ストーリー / 問いかけ。同じ要素が3つ以上連続すると単調。

|評価|基準|
|-|-|
|A|多様な要素が交互に配置され、飽きが来ない|
|B|バリエーションはあるが、一部で単調なセクションがある|
|C|要素の種類が偏っている（説明ばかり等）|
|D|変化がなく単調|

### 軸12: 感情の起伏

|評価|基準|
|-|-|
|A|感情の波が明確で、共感・驚き・納得のポイントが複数ある|
|B|感情が動くポイントはあるが、大部分は平坦|
|C|ほぼ一本調子で、教科書的|
|D|感情的な接点がない|

### 軸13: 記憶に残るフレーズ

|評価|基準|
|-|-|
|A|複数の印象的なフレーズがあり、内容と自然に結びついている|
|B|1-2個の良いフレーズがある|
|C|フレーズはあるが印象に残りにくい|
|D|記憶に残る表現がない|

### 軸14（オプション）: 映像/スライドとナレーションの整合

Project Integration で `Visual Assets` が宣言されている場合のみ評価する。
未宣言ならスキップし、その旨をレポートに記載する。

|評価|基準|
|-|-|
|A|映像とナレーションが完全に同期し、相互補強している|
|B|概ね一致しているが、1-2箇所でタイミングのずれや不一致がある|
|C|部分的に映像とナレーションが乖離している|
|D|映像とナレーションが別々の内容を伝えている|

全資産の確認が現実的でなければ、各幕からサンプリング（2-3シーン）で代用する。

## カテゴリD: 説得 (軸15-18)

### 軸15: 信頼性の構築

|評価|基準|
|-|-|
|A|実績と実例で自然に信頼性が確立されている|
|B|信頼性はあるが、もう少し具体的な根拠があると強い|
|C|信頼性の根拠が弱いか、自慢調になっている|
|D|なぜこの人の話を聞くべきかが不明|

### 軸16: 反論への対処

|評価|基準|
|-|-|
|A|主要な反論を先回りで解消し、限界も正直に提示|
|B|いくつかの反論に対処しているが、見落としがある|
|C|反論への対処が薄い|
|D|反論を完全に無視している|

### 軸17: テイクアウェイ（持ち帰り）

|評価|基準|
|-|-|
|A|具体的で実行可能なアクションが明確、フックと円環構造|
|B|アクションは示されているが、やや抽象的|
|C|結論はあるが、具体的な行動に落ちていない|
|D|結論が不明確、または欠落|

### 軸18: タイトルとの一致

|評価|基準|
|-|-|
|A|タイトルの約束を完全に果たし、期待通りの内容|
|B|概ね一致しているが、タイトルから期待する一部が薄い|
|C|タイトルと本編にギャップがある|
|D|タイトル詐欺、または内容と無関係|

## 総合判定

- **基本ルール**: 18軸の最頻値（軸14未評価時は17軸）
- **特別ルール**: 正確性（軸5）がDなら総合もD（事実誤認は致命的）
- **A判定の目安**: A×12以上（評価対象軸の2/3）
- **B判定の目安**: A×8以上 かつ D×0
- **完璧を求めない**: A×12以上なら「出して良い品質」と判断する
