# Evaluation Axes

各軸はA/B/C/Dの4段階で採点し、具体的な根拠と改善案を添える。**各評価は、根拠となったナレーションの一文を必ず引用する**（引用できない評価はそれ自体が要修正のサイン）。

軸は1つのゲート軸＋4カテゴリ（構造/内容/伝達/説得）＋条件付き軸で構成される。各軸末尾に、その軸が依拠する名のあるフレームワーク（出典）を示す。出典の付かない数値・主張は「運用ヒューリスティック」と明記する。

> **言語前提**: 文字数・話速（wpm）の数値は日本語の口語を前提にした目安であり、他言語には転用しない。char数の帯は読解研究を口語日本語に外挿したもので、厳密な根拠ではない（ソフトな帯として使う）。

## ゲート軸: 中心メッセージ（throughline）

**「この発表の主張を一文で言えるか。全セクションがそこに結びついているか。」**

最も合意の強い構造原則（Anderson の throughline ≤15語／AAC&U Oral Communication の "Central Message"／Minto の answer-first／Heath "Simple"）。軸2（SCQAの形）・軸3（節のつながり）・軸18（タイトル一致）はこれを部分的にしか見ていない。**これはゲート軸**: C以下なら総合はCで頭打ち。

|評価|基準|
|-|-|
|A|15語前後で言い切れる1つの中心メッセージがあり、各セクションが目に見えてそこに結びつく|
|B|中心メッセージは特定できるが、汎用的 or 複数文に渡る|
|C|主題/タイトルしかなく、中心メッセージは聞き手が推測するしかない|
|D|話題の羅列で、1つの主張がない|

出典: Chris Anderson *TED Talks*（throughline） https://www.ted.com/talks/chris_anderson_ted_s_secret_to_great_public_speaking ／ AAC&U Oral Communication VALUE Rubric（Central Message） https://www.aacu.org/value/rubrics/value-rubrics-oral-communication ／ Minto Pyramid（answer-first）／Heath *Made to Stick*「Simple」。

## カテゴリA: 構造 (軸1-4)

### 軸1: フック（最初の30秒）

視聴者が「続きを見よう」と思うか。効果的なパターン: 問いかけ（経験に直接訴える）／意外な数字（常識を覆す）／ストーリー（具体的エピソード）。Anderson は「冒頭の挨拶・自己紹介・アジェンダで時間を使わず、最初の一文で好奇心の隙間を開ける」ことを推奨。

|評価|基準|
|-|-|
|A|視聴者が即座に自分ごと化できる問いかけ or 意外性。冒頭から好奇心の隙間が開く|
|B|興味は引くが、自分ごと化までは至らない|
|C|一般的な導入（「今日は〜について話します」）|
|D|フックなし、いきなり本題 or 挨拶・アジェンダで埋める|

出典: Anderson TED 流の冒頭設計 https://mannerofspeaking.org/2016/05/24/the-ted-secret-to-great-public-speaking/ ／ Heath "Unexpected"（好奇心の隙間）。

### 軸2: ナラティブアーク（SCQA構造）

全体がストーリーとして成立しているか。**ラベルが揃っているかではなく、SOURCEのパターン（緊張→解決）が成立しているか**で見る。SCQA: Situation（前提）→ Complication（障害＝解決前の本物の緊張）→ Question（問い）→ Answer（解答）。Story Spine の「ある日（Until one day）」の転換が骨。**転換のない区間は説明であってストーリーではない**（その区間はCで頭打ち）。struggle（うまくいかなかったこと）が共感と説得力を生む。

|評価|基準|
|-|-|
|A|Situation→Complication（解決前に本物の緊張）→明示的なQuestion→Answer が揃い、struggle/転換の beat がある|
|B|SCQAの形はあるが、struggleか転換が弱い|
|C|問題→解決はあるが緊張感が薄い／いきなりAnswerに飛ぶ|
|D|情報の羅列で、ストーリーがない|

出典: Minto SCQA https://modelthinkers.com/mental-model/minto-pyramid-scqa ／ Kenn Adams Story Spine（"Until one day"の転換） https://www.sessionlab.com/methods/story-spine 。

### 軸3: 論理の流れ（ゲート軸）

各セクションが前のセクションの上に積み重なっているか。節のつなぎ（signpost / preview / summary）が機能しているか。**ゲート軸**（論理がDなら総合に効く）。

|評価|基準|
|-|-|
|A|全セクションが自然に繋がり、順序に必然性があり、節のつなぎが明示される|
|B|概ね良いが、1-2箇所で唐突な転換がある|
|C|各セクション単体はOKだが、全体の流れが弱い|
|D|順序が恣意的、前後を入れ替えても違和感がない|

出典: connective statements / signposts https://fscj.pressbooks.pub/publicspeaking/chapter/connective-statements-signposts-and-transitions/ ／ Minto top-down。

### 軸4: ペーシング（情報密度の配分）

> [!IMPORTANT]
> 旧基準「密度が均一＝A」は**誤り**。研究は意図的な緩急（contrast）と息継ぎを評価する。均一に詰まっている＝単調、も減点対象。

高密度/低密度の区間が交互にあり、息継ぎ（breather）の beat があるか。要点の前後で速度を落としているか。最重要点を冒頭と終盤に置いているか（primacy-recency）。

|評価|基準|
|-|-|
|A|高密度と低密度が緩急をつけて交互に来る。要点は速度を落とし、最重要点が冒頭と終盤にある|
|B|概ね良いが、緩急が一部弱い|
|C|前半と後半で密度差が大きい、または全体に平坦|
|D|全体的に詰め込みすぎ or スカスカ|

出典: Sweller Cognitive Load Theory / segmenting https://journals.physiology.org/doi/full/10.1152/advan.00109.2016 ／ Serial Position Effect https://www.ncbi.nlm.nih.gov/pmc/articles/PMC11284078/ ／ Duarte「contrast」 https://www.duarte.com/blog/ultimate-guide-to-contrast/ 。

## カテゴリB: 内容 (軸5-8)

### 軸5: 正確性（ゲート/オーバーライド軸）

> [!IMPORTANT]
> 軸5だけは妥協しない。事実誤認は1つでもあれば指摘する。**軸5がDなら総合もD**（特別扱い）。

|評価|基準|
|-|-|
|A|全て正確、出典付きデータと意見の区別も明確|
|B|概ね正確だが、1-2箇所で微妙な不正確さ|
|C|重要でない部分に不正確さ|
|D|核心部分に事実誤認|

出典: Heath "Credible"（信頼性の作り方）／一般的な出典・ethos規範。

### 軸6: 網羅性（ゲート候補軸）

参照ソースの重要情報がカバーされているか。

|評価|基準|
|-|-|
|A|コアメッセージを全てカバーし、過不足ない|
|B|主要な内容はカバー、1-2点の補足が欲しい|
|C|いくつかの重要ポイントが欠けている|
|D|核心的な情報が欠落|

出典: 参照ソース突合（`--source`）の網羅性。

### 軸7: 具体性

主張に具体例・数字・固有名詞が伴っているか。

|評価|基準|
|-|-|
|A|全主張に具体例・数字・固有名詞が伴い、解像度が高い|
|B|大部分は具体的だが、1-2箇所で抽象的|
|C|具体例が散発的で抽象論が多い|
|D|ほぼ抽象論で根拠や実例がない|

出典: Heath "Concrete"（具体・画像優位性） https://heathbrothers.com/books/made-to-stick/ 。

### 軸8: 冗長性（削れる繰り返し）

**取り除いても内容が変わらない繰り返し**に限定して見る（密度の配分は軸4へ）。意図的な反復（効果的な繰り返し）は減点しない。

|評価|基準|
|-|-|
|A|削れる冗長な繰り返しなし、意図的な反復は効果的|
|B|1-2箇所で冗長だが全体には影響しない|
|C|複数箇所で同じ内容を繰り返す|
|D|大幅にカットしても内容が変わらない|

出典: Mayer Coherence Principle（不要物を削る） https://www.cambridge.org/core/books/cambridge-handbook-of-multimedia-learning 。

## カテゴリC: 伝達 (軸9-14)

### 軸9: 明瞭さ（聞き取りやすさ）

抽象→具体の流れが一貫しているか。口語の聞き取りやすさ。会話的な語り口（一人称・話しかけ）か。

|評価|基準|
|-|-|
|A|全体的に平易、抽象→具体が一貫、会話的な語り口|
|B|概ね明瞭だが、1-2箇所で理解に時間がかかる|
|C|部分的に難解で具体例が不足|
|D|全体的にわかりにくい|

口語の目安（**日本語前提・ソフトな帯**）: 一文は平均15〜20語／おおむね40〜50字以内、難所はさらに短く。これは読解研究を口語日本語に外挿した運用ヒューリスティックで、厳密な閾値ではない。会話的な語り口は Mayer の Personalization/Voice 原理。出典: Mayer Personalization https://www.digitallearninginstitute.com/blog/mayers-principles-multimedia-learning 。

### 軸10: 視聴者フィット（知識の呪い）

想定視聴者は Project Integration の `Audience` で宣言、未宣言なら参照ソースから推定。**「知識の呪い」**サブチェック（話し手には自明でも聞き手に未説明の用語/飛ばした手順）を数で見る。

|評価|基準|
|-|-|
|A|想定視聴者の非専門家が全手順を追え、初出用語は全て説明済み|
|B|概ね適切だが、1-2箇所で説明不足の専門用語|
|C|未説明の用語/飛ばした手順が2-3箇所|
|D|視聴者が持たない専門性を前提にしている|

出典: 知識の呪い＝Camerer/Loewenstein/Weber 1989 https://en.wikipedia.org/wiki/Curse_of_knowledge ／ AAC&U "Language"。※Newton 1990 の「タッパーと聞き手」実験は**別の**例示（1989年論文とは別物）として扱い、引用をまとめない。

### 軸11: エンゲージメント（飽きさせない）

飽きを防ぐ要素: 具体例 / データ / デモ / 比較 / ストーリー / 問いかけ。同じ要素が3つ以上連続すると単調。

|評価|基準|
|-|-|
|A|多様な要素が交互に配置され飽きが来ない|
|B|バリエーションはあるが一部単調|
|C|要素の種類が偏る（説明ばかり等）|
|D|変化がなく単調|

出典: Mayer Signaling ／ 注意の途切れ（Bradbury 2016＝「10分で集中が切れる」は俗説。固定時間ではなく再喚起の頻度で見る） https://journals.physiology.org/doi/full/10.1152/advan.00109.2016 。

### 軸12: 緊張のオシレーション（観察可能な構造として）

> 主観の「面白さ」ではなく、**観察できる構造**で見る（QAの「主観で評価しない」と整合）。

Duarte の「what-is（現状）／what-could-be（あるべき姿）」の振れが繰り返し現れるか。引用できる beat を根拠にする。

|評価|基準|
|-|-|
|A|現状⇔あるべき姿の振れが複数あり、引用できる beat がある|
|B|振れはあるが大部分は平坦|
|C|ほぼ一本調子で教科書的|
|D|緊張の振れがない|

出典: Duarte Sparkline / Contrast https://www.duarte.com/blog/ultimate-guide-to-contrast/ ／ Heath "Emotional"。

### 軸13: 記憶に残るフレーズ（STARモーメント）

> 主観ではなく、**繰り返せる引き締まった一言（sound-bite）が在るか**で見る。

|評価|基準|
|-|-|
|A|繰り返せる引き締まったフレーズが複数あり、内容と自然に結びつく（引用可）|
|B|1-2個の良いフレーズ|
|C|フレーズはあるが印象に残りにくい|
|D|記憶に残る表現がない|

出典: Duarte STAR（Something They'll Always Remember）／ tricolon・三の法則。

### 軸14（オプション）: 映像/スライドとナレーションの整合（タイミング側のみ）

Project Integration で `Visual Assets` が宣言されている場合のみ評価。未宣言ならスキップしレポートに明記。**この軸はナレーションと画面内容の同期（タイミング）だけを見る**。スライド単体のレイアウト/コントラスト/フォント等の視覚品質は slide-visual-reviewer に委譲（同じ対象を二重採点しない）。

|評価|基準|
|-|-|
|A|映像とナレーションが同期し相互補強|
|B|概ね一致だが1-2箇所でタイミングのずれ|
|C|部分的に映像とナレーションが乖離|
|D|映像とナレーションが別々の内容|

全資産の確認が現実的でなければ各幕から2-3シーンをサンプリング。

## カテゴリD: 説得 (軸15-18)

### 軸15: 信頼性の構築

|評価|基準|
|-|-|
|A|実績と実例で自然に信頼性が確立|
|B|信頼性はあるが、もう少し具体的な根拠が欲しい|
|C|信頼性の根拠が弱い or 自慢調|
|D|なぜこの人の話を聞くべきかが不明|

出典: Aristotle ethos https://plato.stanford.edu/entries/aristotle-rhetoric/ ／ Cialdini「権威」。

### 軸16: 反論への対処（接種理論）

反論を**二面的・反駁的**に扱えているか（McGuire の接種理論）。

|評価|基準|
|-|-|
|A|最も強い反対意見を最も好意的な形（steelman）で挙げ、かつ反駁している|
|B|反論は挙げるが弱い/藁人形になっている|
|C|反論への対処が薄い（ほのめかす程度）|
|D|明らかな反論を無視|

出典: McGuire 接種理論（二面的反駁メッセージ） https://en.wikipedia.org/wiki/Inoculation_theory 。※「steelman / 寛容の原則」は俗語的なヒューリスティック（学術理論ではない）として扱い、研究フレームワークとして提示しない。

### 軸17: テイクアウェイ＋締め

(1) 具体的で実行可能な持ち帰り、AND (2) **名のある締め方**（Anderson の Vision / Encapsulation / Symmetry-callback / Lyric）と、書き切った最後の一文。Symmetry-callback は冒頭フックとゲート軸（throughline）に呼応させる。

|評価|基準|
|-|-|
|A|具体的アクションが明確 ＋ 名のある締め方＋書き切った最終文（フック/中心メッセージと円環）|
|B|アクションは示すがやや抽象的、締めが弱い|
|C|結論はあるが行動に落ちない、問いかけで尻すぼみ|
|D|結論が不明確 or 欠落|

出典: Anderson 締めの型 https://mannerofspeaking.org/2016/05/24/the-ted-secret-to-great-public-speaking/ ／ Peak-End（Kahneman/Fredrickson） https://en.wikipedia.org/wiki/Peak%E2%80%93end_rule 。

### 軸18: タイトルとの一致

|評価|基準|
|-|-|
|A|タイトルの約束を完全に果たす|
|B|概ね一致だが期待の一部が薄い|
|C|タイトルと本編にギャップ|
|D|タイトル詐欺 or 内容と無関係|

## 条件付き軸: 時間予算（軸19・runtimeが宣言された時のみ）

Project Integration で持ち時間（分）が宣言されたときのみ評価（軸14と同じく条件付き）。未宣言ならスキップして明記。原稿語数 ÷ 話速で持ち時間に収まるか。**話速・字数は日本語前提**（おおむね 1分 ≒ 300〜350字／英語なら ~130〜160 wpm）。

|評価|基準|
|-|-|
|A|余裕を持って収まり、要点で速度を落とせる|
|B|ほぼ収まる（±10%以内）|
|C|10〜25%超過（巻きが要る）|
|D|25%超過（時間内に終わらない）|

出典（運用目安）: 話速ガイド https://virtualspeech.com/blog/average-speaking-rate-words-per-minute ／ https://tfcs.baruch.cuny.edu/speaking-rate/ 。※QAの「動画の長さ自体は評価しない」は**恣意的な尺**を指すもので、宣言された持ち時間への収まりとは別。条件付きなので矛盾しない。

## 総合判定（ゲート＋しきい値方式）

> 旧「最頻値（MODE）」は誤り。MODEは**同じ軸を複数の評価者で合議する**ための集計で、異種の軸をまたいで使うと、正確性以外の単独Dを多数決で隠し、同点時の決着も未定義になる。以下のゲート＋しきい値方式に置き換える。

**ゲート軸**（これらだけが総合を頭打ちにする）: `throughline`（ゲート）／`軸3 論理`／`軸5 正確性`（オーバーライド）／`軸6 網羅性`。装飾寄りの軸（軸12 緊張・軸13 フレーズ）はAカウントには寄与するが**ゲートにはしない**。

判定（明示カウントで決める。Aカウントは評価対象軸の中のA個数）:

- **D**: いずれかのゲート軸＝D、または 軸5＝D。
- **C**: 正確性以外に1つでもD、または いずれかのゲート軸＝C（throughlineのC以下を含む）。
- **B**: Aカウント ≥ 8、Dゼロ、かつ全ゲート軸 ≥ B。
- **A**: Aカウント ≥ 12、Dゼロ、かつ全ゲート軸 ≥ B。

**根拠アンカー（必須ルール）**: すべての評価は、根拠となった原稿の一文を引用する。Aには引用できる模範文が必須。引用できる根拠の無い評価は、それ自体を要修正として下げる。（これは本スキル既存の「引用必須」方針と一般的な evidence-grounded 評価の実践に基づく。特定の名前のフレームワークには帰属させない。）

**インフレ防止**: Aは理想ではなく実在の最良例に対して付ける。Bを「ちゃんと出せる/競争力ある」現実的な合格ラインとして書き、良い原稿の多くがBに落ちるよう運用する。「完璧を求めない」: Aカウント≥12なら「出して良い品質」と判断。
